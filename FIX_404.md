# 404 错误修复指南

## 问题原因

GitHub Pages 显示 404 是因为 Vue 项目缺少 `base` 路径配置。

## 已完成的修复

✅ 已在 `vite.config.js` 中添加：
```javascript
base: '/low-carbon-app/',
```

## 需要手动完成的操作

由于网络不稳定，请手动推送代码：

### 步骤 1: 打开终端

在项目目录下打开终端（PowerShell 或 CMD）

### 步骤 2: 推送代码

执行以下命令：
```bash
git push origin master
```

### 步骤 3: 等待部署

1. 访问 https://github.com/onezms/low-carbon-app/actions
2. 等待工作流显示绿色 ✓

### 步骤 4: 访问网站

部署成功后，访问：
```
https://onezms.github.io/low-carbon-app
```

## 如果还是 404

等待 2-5 分钟，GitHub Pages 需要时间更新。

## 清除缓存

如果还是显示 404，尝试：

1. **清除浏览器缓存**
2. **使用无痕模式访问**
3. **强制刷新** (Ctrl+F5 或 Cmd+Shift+R)

## 检查部署

确认 dist 目录已正确部署：

1. 访问 https://github.com/onezms/low-carbon-app/settings/pages
2. 查看部署状态
3. 确认 Custom domain 设置为 `dtshgj.top`

## 需要帮助？

如果推送后还是 404，请告诉我：
1. 工作流是否显示绿色 ✓
2. 访问哪个 URL 显示 404
3. 错误页面的完整内容
